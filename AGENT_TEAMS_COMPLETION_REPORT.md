## 🎯 三大用户贡献

### 贡献 1：去中心化协作架构
**洞察**："为何团队中缺少 leader 一类的编排者"
**实现**：从单一 Leader 协调 → 对等角色互补生态
**价值**：更鲁棒、更灵活、符合现代敏捷理念

### 贡献 2：内置批判思维机制
**洞察**："为了保证发散性思考,是否有必要加入假设性竞争代理"
**实现**：从单一批判者 → 所有角色间假设竞争对话
**新增角色**：
- Devil's Critic（魔鬼式批判者）
- Risk Analyst（风险分析师）
- Issue Improver（问题改进专员）
- New Feature Specialist（新特性探索专员）
**价值**：避免群体思维、激发创新、提高决策质量

### 贡献 3：5-7 人黄金法则
**洞察**："通常对于中型以下项目,团队成员维持在 5-7 人规模"
**实现**：
- 明确的团队规模指导原则
- 4 种验证的配置模板
- 基于"两个披萨"理论和邓巴数理论
**价值**：沟通效率最优、决策迅速、实践验证

## 🚀 如何使用

### 立即可用（三种方式）

#### 方式 1：交互式（最简单）
```
直接告诉 Claude："创建一个 6 人团队做产品调研"
Claude 自动配置并启动
```

#### 方式 2：声明式（最灵活）
```yaml
# 创建 .team.yaml
name: my-team
team_size: 6
agents:
  - name: architect
    role: 构建者
  - name: critic
    role: 批判者
    critical_thinking_style:
      skepticism: 0.9
  # ... 其他 agents

# 运行
/team-launch my-team
```

#### 方式 3：模板式（最快）
```yaml
# 使用内置模板
template: fullstack-research
parameters:
  - name: tech_stack
    value: "microservices"
```

### 推荐工作流

1. **开始**：阅读 `QUICKSTART.md`（5 分钟了解）
2. **选择**：从 4 种最佳配置中选择适合的（5/6/7 人）
3. **定制**：根据需求调整角色和任务
4. **执行**：使用交互式或声明式启动
5. **监控**：使用 `/team-status` 和 `/team-logs`
6. **迭代**：根据实际效果优化配置

## 📖 关键特性

### 1. 对等协作 + 假设竞争

```
┌────────────────────────────────────────┐
│         假设竞争对话                 │
├────────────────────────────────────────┤
│  [构建者] "我建议 React 19"       │
│  ↓                                  │
│  [批判者] "如果 Server Components    │
│             集成困难怎么办？"           │
│  ↓                                  │
│  [改进者] "可以分阶段迁移"         │
│  ↓                                  │
│  [整合者] "综合：混合方案"        │
└────────────────────────────────────────┘
```

### 2. 四类角色生态

| 类型 | 典型角色 | 核心思维 | 团队占比 |
|------|---------|---------|---------|
| 构建者 | Solution Architect | "如何做？可能性？" | 30-40% |
| 批判者 | Devil's Critic | "如果错？风险？" | 20-30% |
| 改进者 | Issue Improver | "更好？创新？" | 20-30% |
| 整合者 | Decision Facilitator | "如何整合？" | 10-20% |

### 3. 团队规模黄金法则

| 人数 | 配置 | 适用 | 优缺点 |
|------|------|------|---------|
| **5 人** | 2构建+1批判+1改进+1整合 | 快速交付 | ✅最快<br>❌单一 |
| **6 人** ⭐ | 2构建+2批判+1改进+1整合 | 平衡创新 | ✅完整<br>✅推荐 |
| **7 人** | 2构建+2批判+2改进+1整合 | 深度探索 | ✅最广<br>❌复杂 |
