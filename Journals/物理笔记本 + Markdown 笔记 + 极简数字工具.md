他在这个视频里讲的是「当前项目/工作的整体生产力工作流」，核心是：**物理笔记本 + Markdown 笔记 + 极简数字工具** 组合起来，用最少的软件支撑完整闭环。[[youtube](https://www.youtube.com/watch?v=x22k3csfJCo&t=420s)]​

## 数字部分（当前项目相关）

- 笔记形式：所有工作相关的笔记都用 Markdown 文件管理，主要在 Neovim 里编辑，强调“编辑器可换，文件格式不绑死平台”。​
    
- 组织方式：
    
    - 每个主题/项目有一份结构化的 note，例如视频项目就有「ideas → outline → recording → thumbnail/title/description → upload」这样的阶段分区。​
        
    - 使用 `render-markdown.nvim` 之类的插件渲染标题、列表等，让结构更清晰，便于在一个大 note 里快速扫状态。​
        
- 同步：用 `rsync` 做双向同步（桌面机 ↔ ThinkPad），并把同步操作直接集成进 dmenu notes 脚本菜单：
    
    - 在 dmenu 菜单里可以模糊搜索/创建笔记。
        
    - 菜单里直接有「sync notes to / from」选项，一键触发 rsync，保证不同机器上的当前项目笔记保持一致。​
        

## 物理笔记本（当前项目的日常驱动）

- 左页：
    
    - 实体 to‑do list（当天/几天的任务），按需标日期。
        
    - 顶部有一个“长期记忆区”，写需要在未来几周甚至几个月持续记住的事项，翻页时也会持续抄过去。​
        
    - 右半部分用来画大结构：比如当前项目的 A→B→C→D 流程、子任务树（D 下再分 E/F/G），更适合用图形/框图“把脑子摊开”。​
        
- 右页：
    
    - 每天回答“今天有没有完成必须完成的事？如果没有，原因是什么？明天/下周要怎么改善？”，做工作反思。​
        
    - 记录当天特别好/特别糟的事情，帮助把情绪写出去、从脑子里清空。​
        
    - 习惯追踪（更偏个人，但也与工作表现相关）：手工画 habit tracker，支持“部分完成”的标记，而不是只分完成/没完成。​
        

## 工具使用的原则（对当前项目的影响）

- 极简原则：不要为了“生产力”装一堆工具，而是用极少数真正解决问题的工具，把时间花在项目本身而不是维护系统。​
    
- 日历与时间追踪：
    
    - 日历主要仍然是实体纸质日历，辅以 Calcure / calcurse 这类 TUI 日历工具做少量数字管理。​
        
    - 用 Zeit 这类 CLI 工具追踪自己在不同项目/任务上的时间，生成图表，替代手工记时，让你更清楚当前项目到底投入了多少精力。​
