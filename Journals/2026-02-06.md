## Sourcegraph 代码搜索指南

### 什么是 Sourcegraph？

Sourcegraph 是一个强大的代码搜索和代码智能平台，支持跨多个代码仓库进行语义搜索。它比传统的 grep/ack/ag 更智能，能理解代码结构。

### 核心特性

1. **跨仓库搜索** - 一次性搜索多个代码仓库
2. **语义搜索** - 理解代码语法，而非纯文本匹配
3. **代码智能** - 提供跳转定义、查找引用等功能
4. **正则表达式支持** - 强大的模式匹配

### 搜索语法

#### 基础搜索

```
# 简单关键词搜索
function_name

# 多关键词（AND 关系）
class_name method_name
```

#### 语法过滤

```
# 只搜索特定语言
lang:python def test_

# 只搜索文件路径
path:src/api/

# 只搜索特定文件名
file:*.go
```

#### 结构化搜索

```
# 搜索函数定义
r:func\s+(\w+)\s*\(

# 搜索类定义
r:class\s+(\w+)

# 搜索 import 语句
lang:python r:from\s+(\w+)\s+import
```

#### 模糊搜索

```
# 不区分大小写
search_term lang:javascript -case-insensitive

# 通配符
func_*_test

# 正则表达式（使用 r: 前缀）
r:\b(get|set)\w*\(
```

### 常用搜索模式

#### 1. 查找 API 调用

```regex
# 查找特定函数的所有调用
r:fetch\s*\(.*?\)

# 查找特定 API 端点
path:api/ r:\/api\/v1\/\w+
```

#### 2. 查找配置项

```regex
# 环境变量引用
r:process\.env\.\w+

# 配置文件
file:config.*\.(json|yaml|toml)
```

#### 3. 查找错误处理

```regex
# try-catch 块
lang:javascript r:try\s*\{[\s\S]*?\}\s*catch

# 错误日志
lang:python r:logger\.(error|exception|warning)
```

#### 4. 查找 TODO/FIXME

```
TODO lang:javascript
FIXME lang:python
HACK
```

#### 5. 查找测试

```regex
# 测试函数
lang:python r:def test_\w+

# 测试断言
r:assert(Equals|True|False)
```

### 高级技巧

#### 1. 组合搜索

```
# 搜索 Python 中的测试函数
lang:python path:test/ r:def test_

# 排除某些文件
lang:go -path:vendor/ -path:.git/ r:func\s+(\w+)
```

#### 2. 上下文搜索

```
# 搜索函数及其注释
r:\/\*\*[\s\S]*?\*\/\s*func\s+\w+

# 搜索变量声明
lang:javascript r:(const|let|var)\s+(\w+)\s*=
```

#### 3. 代码重构查找

```regex
# 查找可能重复的代码块
r:if\s*\([^)]*\)\s*\{\s*return\s+true\s*\}

# 查找长函数
r:func\s+\w+\([^)]*\)\s*\{[\s\S]{500,}
```

### 实用搜索示例

#### 查找未使用的导入
```regex
lang:python r:^import\s+(\w+)
```

#### 查找硬编码的密钥
```regex
-password
-secret
-token
-api_key
```

#### 查找 SQL 注入风险
```regex
r:query\s*=\s*['"].*\$\{.*\}
```

#### 查找性能瓶颈
```regex
r:for\s*\([^)]*;\s*[^;]*\+\+[^;]*\)\s*\{
r:forEach\([\s\S]{0,100}\)
```

### 搜索结果优化

1. **使用更具体的路径过滤** - `path:src/controllers/api/`
2. **限定语言** - `lang:typescript`
3. **使用正则精确匹配** - 避免 false positive
4. **排除测试文件** - `-path:test/ -path:*test*`

### 最佳实践

✅ **DO**
- 使用路径过滤缩小搜索范围
- 优先使用结构化搜索（r: 语法）
- 保存常用搜索模式
- 组合多个条件精确搜索

❌ **DON'T**
- 使用过于宽泛的关键词（如 "data"）
- 忽略语言过滤
- 在大仓库中盲目全局搜索
