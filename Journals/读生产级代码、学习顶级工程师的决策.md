## 核心逻辑

优秀后端工程师的本质是**「在真实系统中建立工程判断力」**。GitHub/grep.app 的价值不是找答案，而是**读生产级代码、学习顶级工程师的决策**。

---

## 四个训练阶段

### 阶段一：建立「代码阅读基线」（第 1-2 个月）

**目标**：能看懂生产级项目的结构和核心路径

**每日任务（30-45 分钟）**

```
# 1. 找高质量项目作为阅读素材
GitHub Search:
language:python stars:>3000 topic:backend archived:false pushed:>2024-01-01

language:go stars:>2000 topic:api archived:false

# 2. 进入仓库后，优先读这些文件
- README → 理解设计目标
- ARCHITECTURE.md / docs/ → 理解系统边界
- main.go / app.py / Program.cs → 入口点
- 核心 handler/controller 层
```

**训练动作**： 每周选 1 个明星项目，用 grep.app 搜它的核心概念：

```
# 例如研究「错误处理」这个主题
repo:gin-gonic/gin Error
repo:fastapi/fastapi HTTPException
repo:expressjs/express next(err)

# 对比 3 个项目的实现差异 → 写 100 字总结
```

### 阶段三：研究顶级公司工程实践（第 6-8 个月）

**目标**：理解「工业级」和「教程级」代码的本质差距

**固定研究对象清单**

```
# 精读这些组织的代码
org:stripe          # 支付系统、API 设计标杆
org:cloudflare      # 高性能网络服务
org:hashicorp       # 基础设施工具（Terraform/Vault）
org:grafana         # 可观测性
org:planetscale     # 数据库即服务
```

**每个组织的研究方法**

```
# Step 1：找他们解决某问题的方式
grep.app:
repo:stripe/stripe-go error handling

# Step 2：找对应的测试代码（测试是最好的文档）
grep.app:
repo:hashicorp/vault TestAcc
path: **/*_test.go

# Step 3：看 PR 和 Issue（GitHub Search）
repo:grafana/grafana is:pr is:merged label:performance

# Step 4：提炼工程决策，记录「他们为什么这样设计」
```

### 阶段四：建立「工程判断力」训练循环（第 9-12 个月）

**每周一个「对比研究」**

```
选一个后端概念，找 3 种不同语言的实现对比：

例：「优雅关机（Graceful Shutdown）」

grep.app 搜：
signal.NotifyContext SIGTERM lang: Go
signal(SIGTERM) graceful lang: Python  
process.on('SIGTERM') lang: JavaScript

→ 对比：各自的信号处理时机、超时控制、连接排空策略
→ 输出：自己实现一个，并能解释每一行的工程意图
```

**每月一个「源码拆解」**

选一个你用过的库，读它的核心实现：

```
# 例：研究 FastAPI 的依赖注入实现
grep.app:
repo:tiangolo/fastapi Depends solve_dependencies
path: fastapi/

GitHub Search:
repo:tiangolo/fastapi is:issue "dependency injection" is:closed
```

## 配套习惯

|习惯|具体做法|
|---|---|
|**代码笔记**|每次搜索发现好代码，记录链接 + 一句话说明「好在哪里」|
|**对比写作**|每月写一篇「X 问题的 3 种实现对比」|
|**Issue 追踪**|关注顶级项目的 Issue，看真实 bug 如何被发现和修复|
|**PR 阅读**|读 merged PR 比读代码更有价值，能看到「改动前后的权衡」|

---

## 一句话总结

**grep.app 用来找「怎么写」，GitHub Search 用来找「为什么这样设计」**——两者结合，本质是把顶级工程师的决策过程变成你自己的工程直觉。
