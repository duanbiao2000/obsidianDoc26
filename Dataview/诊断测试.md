# Dataview 诊断测试

> 逐步测试 Dataview 查询，找出问题所在。

## 📝 测试1: 基础查询（应该工作）

```dataview
TABLE title, file.ctime
FROM ""
LIMIT 5
```

**预期**: 显示5个笔记及其创建时间

---

## 📝 测试2: 排序查询（应该工作）

```dataview
TABLE title, file.ctime
FROM ""
SORT file.ctime DESC
LIMIT 5
```

**预期**: 显示最近创建的5个笔记

---

## 📝 测试3: date() 函数测试

```dataview
TABLE WITHOUT ID
  date(today) AS "今天",
  date(now()) AS "现在"
FROM ""
LIMIT 1
```

**预期**: 显示今天的日期和当前时间

---

## 📝 测试4: dur() 函数测试

```dataview
TABLE WITHOUT ID
  date(today) - dur(7 days) AS "7天前",
  date(today) + dur(7 days) AS "7天后"
FROM ""
LIMIT 1
```

**预期**: 显示7天前和7天后的日期

---

## 📝 测试5: 简单日期比较（应该工作）

```dataview
TABLE title, file.ctime
FROM ""
WHERE file.ctime >= date("2026-03-01")
SORT file.ctime DESC
```

**预期**: 显示2026-03-01之后创建的所有笔记

---

## 📝 测试6: 使用 dateformat 的日期比较

```dataview
TABLE title, file.ctime, dateformat(file.ctime, "yyyy-MM-dd")
FROM ""
WHERE dateformat(file.ctime, "yyyy-MM-dd") >= "2026-03-01"
SORT file.ctime DESC
```

**预期**: 使用字符串比较查找2026-03-01之后的笔记

---

## 📝 测试7: 使用 dur() 的日期运算

```dataview
TABLE title, file.ctime
FROM ""
WHERE file.ctime >= date(today) - dur(7 days)
SORT file.ctime DESC
```

**预期**: 显示最近7天创建的笔记

---

## 📝 测试8: 检查文件类型

```dataview
TABLE WITHOUT ID
  title,
  typeof(file.ctime) AS "ctime类型"
FROM ""
LIMIT 5
```

**预期**: 所有 ctime 应该是 "date" 类型

---

## 📝 测试9: 查找异常文件

```dataview
TABLE title, file.ctime, typeof(file.ctime)
FROM ""
WHERE typeof(file.ctime) != "date"
```

**预期**: 如果有异常文件（ctime 不是日期类型），会显示出来

---

## 📝 测试10: 检查 Dataview 版本信息

```dataview
TABLE WITHOUT ID
  meta(" Dataview: version") AS "Dataview版本"
FROM ""
LIMIT 1
```

**预期**: 显示 Dataview 插件版本

## 🔍 根据结果诊断

### 如果测试1失败
- **问题**: Dataview 插件可能未启用或损坏
- **解决**: 检查设置 → 社区插件 → 启用 Dataview

### 如果测试2失败
- **问题**: 某些文件的 ctime 可能有问题
- **解决**: 运行测试9查找异常文件

### 如果测试3失败
- **问题**: date() 函数不工作
- **解决**: 更新 Dataview 插件到最新版本

### 如果测试4失败
- **问题**: dur() 函数不工作（可能是旧版本）
- **解决**: 更新 Dataview 插件

### 如果测试5工作，但测试7失败
- **问题**: `date(today) - dur(7 days)` 语法有问题
- **解决**: 使用测试6的 dateformat 方法

### 如果测试9显示异常文件
- **问题**: 某些笔记有损坏的元数据
- **解决**:
  1. 记录异常文件
  2. 检查这些文件的 frontmatter
  3. 修复或删除异常文件

## ✅ 推荐的工作查询

基于测试结果，这是最可能工作的查询：

```dataview
TABLE title, file.ctime
FROM ""
WHERE dateformat(file.ctime, "yyyy-MM-dd") >= dateformat(date(today) - dur(7 days), "yyyy-MM-dd")
SORT file.ctime DESC
```

如果这个也不工作，使用最简单的查询：

```dataview
TABLE title, file.ctime
FROM ""
SORT file.ctime DESC
LIMIT 20
```

然后手动筛选最近的文件。
